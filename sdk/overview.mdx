The Alpha Camera SDK family is a set of official REST clients for the Alpha Camera REST API. Each is published independently and tracks the same versioned API surface.

| Language | Package | Registry | Reference |
|---|---|---|---|
| TypeScript / JavaScript | [`@alpha-sdk/client`](https://www.npmjs.com/package/@alpha-sdk/client) | npm | [TypeScript SDK](/sdk/typescript) |
| Python | [`alpha-sdk-client`](https://pypi.org/project/alpha-sdk-client/) | PyPI | [Python SDK](/sdk/python) |
| Swift | [`AlphaSDK`](https://github.com/jordlee/alpha-sdk-swift) | SwiftPM | [Swift SDK](/sdk/swift) |

## Example apps

Reference apps that use the published SDKs and server package:

- **Next.js / TypeScript:** [`alpha-sdk-typescript`](https://github.com/jordlee/alpha-sdk-typescript)
- **Python notebook / data collection:** [`alpha-sdk-python`](https://github.com/jordlee/alpha-sdk-python)
- **Swift SDK + macOS sample app:** [`alpha-sdk-swift`](https://github.com/jordlee/alpha-sdk-swift)

<Note>
  Every client targets the same REST server. To run that server locally, install [`@alpha-sdk/api`](/web-api/server-package) ‚Äî it ships the native binary, the `camera-server` CLI, and (for Node.js applications) a `ServerManager` class for embedding the server lifecycle. Browser-only or mobile applications connect to a server running elsewhere on the network.
</Note>

## Quick orientation

Every language client follows the same shape:

```
client
  ‚îú‚îÄ‚îÄ server          health, status, logs
  ‚îú‚îÄ‚îÄ cameras         list, connect, disconnect, getConnectionStatus
  ‚îú‚îÄ‚îÄ properties      get, set, getAll, getPriorityKey, setPriorityKey
  ‚îú‚îÄ‚îÄ actions         shutter, halfPress, afShutter, zoom, focusNearFar, movieRec
  ‚îú‚îÄ‚îÄ liveView        enable, disable, getStatus, start, stop, getFrame
  ‚îú‚îÄ‚îÄ sdCard          list, download, downloadThumbnail, downloadScreennail
  ‚îî‚îÄ‚îÄ settings        download, upload, list, importLut
```

The following are intentionally outside the generated client surface ‚Äî see [recipes](#recipes) for the recommended patterns:

- **Server-Sent Events** ‚Äî consume with native `EventSource` (browser), `URLSession.bytes(for:)` (Swift), or `httpx.stream` (Python).
- **Discovery, reconnect, and lifecycle helpers** ‚Äî copy the recipe that fits your application.
