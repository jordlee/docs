For full control without managed state, use the lower-level classes directly. These are the building blocks that CameraManager uses internally.

## CameraClient

Typed HTTP client for all REST endpoints. Works in both Node.js and browsers.

```typescript
import { CameraClient } from 'camera-remote-web-api';

const client = new CameraClient('http://localhost:8080');
```

### Connection

```typescript
// Discover cameras
const { cameras } = await client.listCameras();
const id = cameras[0].id;

// Connect
await client.connectCamera(id, { mode: 'remote' });

// Check status
const status = await client.getConnectionStatus(id);

// Disconnect
await client.disconnectCamera(id);
```

### Properties

```typescript
// Read
const iso = await client.getProperty(id, 'iso');
console.log(iso.data.formatted);  // "ISO 400"

const all = await client.getAllProperties(id);
console.log(all.properties);

// Write
await client.setProperty(id, 'iso', { value: '800' });
await client.setProperty(id, 'aperture', { value: 'f/2.8' });
await client.setProperty(id, 'shutter-speed', { value: '1/250' });

// Priority key (required for remote control)
await client.setPriorityKey(id, { setting: 'pc-remote' });
```

### Actions

```typescript
// Shutter
await client.triggerShutter(id);                              // single shot
await client.triggerShutter(id, { action: 'down' });         // continuous start
await client.triggerShutter(id, { action: 'up' });           // continuous stop

// Focus
await client.halfPress(id);                                   // focus lock
await client.afShutter(id);                                   // AF + capture
await client.focusNearFar(id, { step: -3 });                 // near, medium speed
await client.focusNearFar(id, { step: 7 });                  // far, max speed

// Zoom
await client.controlZoom(id, { direction: 'in', speed: 'normal' });
await client.stopZoom(id);
```

### Live View

```typescript
await client.enableLiveView(id);
await client.startLiveViewStream(id);
const frame = await client.getLiveViewFrame(id);  // ArrayBuffer (JPEG)
await client.stopLiveViewStream(id);
```

### SD Card

```typescript
const files = await client.listSDCardFiles(id, 1);  // slot 1
await client.downloadSDCardFile(id, 1, contentId, fileId);
```

### Settings

```typescript
await client.getSaveInfo(id);
await client.setSaveInfo(id, { path: '/tmp/photos', prefix: 'IMG_', startNo: 1 });
```

### Error Handling

`CameraClient` throws on HTTP 4xx/5xx responses and network errors:

```typescript
try {
  await client.setProperty(id, 'iso', { value: '999999' });
} catch (err) {
  console.error(err.message);  // "Invalid value for property iso"
}
```

## EventStream

Server-Sent Events client for real-time camera notifications. Works in both Node.js and browsers.

```typescript
import { EventStream } from 'camera-remote-web-api';

// All cameras
const events = new EventStream('http://localhost:8080');

events.on('downloadComplete', (data) => {
  console.log('File saved:', data.filename);
});

events.on('propertyChanged', (data) => {
  console.log('Properties changed:', data.codes);
});

events.on('afStatus', (data) => {
  console.log('AF state:', data.state);  // 'focused' | 'unlocked' | 'tracking'
});

events.on('warning', (data) => {
  if (data.code === '0x20011') {
    console.log('Photo captured!');
  }
});

events.on('transferProgress', (data) => {
  console.log(`Transfer: ${data.percent}%`, data.filename);
});

events.on('error', (data) => {
  console.error('SDK error:', data.code);
});
```

### Camera-Specific Events

```typescript
// Filter to a single camera
const camEvents = new EventStream('http://localhost:8080', 'D10F60149B0C');
```

### Cleanup

```typescript
// Remove specific callback
const handler = (data) => console.log(data);
events.on('downloadComplete', handler);
events.off('downloadComplete', handler);

// Close connection
events.close();
```

## LiveViewStream

WebSocket client for continuous JPEG frame streaming at ~15fps. Node.js only — import from `camera-remote-web-api/server`.

```typescript
import { LiveViewStream } from 'camera-remote-web-api/server';

const liveView = new LiveViewStream('ws://localhost:8080', 'D10F60149B0C');

liveView.onFrame((jpeg: ArrayBuffer) => {
  // Each message is a complete JPEG frame
  console.log(`Frame: ${jpeg.byteLength} bytes`);
});

// Clean up
liveView.close();
```

For browser WebSocket usage, see the [Live View endpoint docs](/web-api/live-view#websocket-streaming).

## ServerManager

Low-level binary process manager. Spawns the native CameraWebApp binary and monitors its health. **Use CameraManager instead** unless you need manual control over the binary lifecycle separately from camera connections.

```typescript
import { ServerManager } from 'camera-remote-web-api/server';

const server = new ServerManager({
  port: 8080,
  autoPort: true,   // find next available port if busy
  binaryPath: '/path/to/CameraWebApp',  // optional explicit path
});

await server.start();      // starts binary, waits for HTTP ready
await server.isRunning();  // true
server.getPort();          // 8080 (or next available if autoPort)
server.getPid();           // process ID

await server.stop();       // graceful shutdown
```

### When to use ServerManager directly

- You need to manage the binary independently of camera connections
- You're building a custom connection manager
- You need access to binary stdout/stderr for debugging

```typescript
server.getStdout();  // string[] — binary stdout lines
server.getStderr();  // string[] — binary stderr lines
```
