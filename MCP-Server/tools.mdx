The AI SDK Assistant provides specialized tools that your AI assistant uses to search SDK documentation, code examples, and technical references. Your AI automatically selects the best tool for your question ŌĆö you don't need to specify which tool to use.

<Note>
  Understanding what tools are available helps you ask better questions. For example, if you want C# code, mention it explicitly so the AI uses `search_code_examples` with the right language filter.
</Note>

## Search Tools

| Tool | Description |
|------|-------------|
| `search_sdk` | AI-powered natural language search across all SDK content ŌĆö best for general questions when you don't know the exact API name. |
| `search_hybrid` | Combined semantic and exact-match search ŌĆö faster than `search_sdk`, good when you have some keywords. |
| `search_documentation` | Searches guides, tutorials, and explanatory text only ŌĆö use when you want concepts explained, not code. |
| `search_code_examples` | Finds working C++ or C# code examples ŌĆö accepts a `language` parameter (`cpp`, `csharp`, or `all`). |
| `search_api_functions` | Looks up API function definitions and signatures ŌĆö use when you need parameter details or return types. |
| `search_compatibility` | Searches camera compatibility tables ŌĆö tells you which cameras support which features. |
| `search_exact_api` | Exact-match lookup by API function name (e.g., `SetSaveInfo`) ŌĆö fastest and most precise tool. |
| `search_error_codes` | Looks up SDK error codes (e.g., `CrError_Connect_TimeOut`) ŌĆö returns descriptions and handling examples. |
| `search_warning_codes` | Looks up SDK warning codes ŌĆö helps determine severity and whether warnings can be safely ignored. |
| `search_by_source_file` | Searches within a specific SDK source file ŌĆö use when debugging a particular module (e.g., `CameraDevice.cpp`). |

## Configuration Tools

| Tool | Description |
|------|-------------|
| `get_current_sdk_version` | Returns the active SDK version for the current session and lists all available versions. |
| `set_sdk_version` | Switches between SDK versions (`V1.14.00` or `V2.00.00`) ŌĆö persists for the session. |
| `set_sdk_language` | Filters results by programming language (`cpp`, `csharp`, or `bash`). |
| `set_sdk_type` | Switches between SDK types (`camera-remote` or `ptp`). |
| `set_sdk_subtype` | Filters PTP results by subtype (`ptp-2` or `ptp-3`). |
| `set_sdk_os` | Filters PTP code examples by operating system (`linux`, `windows`, or `cross-platform`). |
| `get_sdk_context` | Shows the current session configuration ŌĆö SDK type, version, language, and active filters. |
| `get_sdk_stats` | Returns database statistics including total indexed content and performance metrics. |

## Quick Reference

| Your Goal | Best Tool | Why |
|-----------|-----------|-----|
| Explore SDK features | `search_sdk` | Understands natural language |
| Find a specific API | `search_exact_api` | Fastest, most precise |
| Get code examples | `search_code_examples` | Language-filtered results |
| Debug an error | `search_error_codes` | Error-specific lookup |
| Learn concepts | `search_documentation` | Text-only explanations |
| Check camera support | `search_compatibility` | Hardware-specific data |
| Balanced search | `search_hybrid` | Speed + accuracy |
