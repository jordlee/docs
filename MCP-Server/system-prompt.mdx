Adding a system prompt to your AI assistant significantly improves response quality when working with the Camera Remote SDK. The system prompt is a comprehensive set of instructions (~450 lines) that teaches the LLM how to effectively use AI SDK Assistant tools and generate accurate, production-ready code.

## Platform-Specific Setup

<Tabs>
  <Tab title="Claude (claude.ai)">
    **Add to Projects:**

    1. Go to [claude.ai](https://claude.ai) and create or open a Project
    2. In the Instructions section, click **+**
    3. Copy the full system prompt from the accordion above
    4. Paste into the Set Project Instructions field
    5. Click **Save Instructions**

    All conversations in that Project will now use the SDK Assistant system prompt.
  </Tab>
  <Tab title="Claude Code">
    **Create a Subagent:**

    1. In the Claude terminal, open the Agents dialogue:

       ```bash
       /agents
       ```
    2. Click **Create new agent**
    3. Select **Project** (current project only) or **Personal** (all projects)
    4. Select **Generate with Claude**
    5. Copy and paste the entire system prompt from the accordion above
    6. Press **Enter** to generate the subagent (this may take a moment)

    The subagent will now use the system prompt for all interactions.

    **Why use a subagent?**

    - Maintains SDK-specific context across sessions
    - Can run tasks in parallel with the main agent
    - Persistent configuration per project or globally

    <Note>
      For simple queries, use AI SDK Assistant with your main agent. For complex SDK development, create a dedicated subagent with the system prompt.
    </Note>
  </Tab>
  <Tab title="Cursor">
    **Add to Rules for AI:**

    1. Open Cursor Settings (`Cmd + ,` on macOS / `Ctrl + ,` on Windows/Linux)
    2. Navigate to **General > Rules for AI**
    3. Copy the full system prompt from the accordion above
    4. Paste it into the **User Rules** text area
    5. Click **Save**

    Alternatively, create a project-level rules file:

    1. Create a `.cursorrules` file in your **project root**
    2. Paste the full system prompt into the file
    3. Save and restart Cursor

    <Tip>
      Project-level `.cursorrules` files are ideal for team collaboration — commit the file to version control so all team members benefit from the system prompt.
    </Tip>
  </Tab>
  <Tab title="ChatGPT">
    <Warning>
      **Not Recommended Unless You Already Have a ChatGPT Subscription**

      Due to OpenAI's unofficial support for MCP servers, ChatGPT has significant limitations:

      - Poor code generation capabilities (pseudo-code only)
      - Unreliable tool usage and may still prefer web search
      - Limited functionality compared to Claude

      We strongly recommend using Claude instead. Only use ChatGPT if you already have an active subscription.
    </Warning>
    **Add to Conversation:**

    1. Copy the full system prompt from the accordion above
    2. Start a new chat (not in a Project — Projects don't support MCP)
    3. Paste the system prompt at the beginning of your conversation
    4. Alternatively, save it as a `.txt` file and upload using the attachment button

    <Note>
      Even with the system prompt, ChatGPT has limited MCP support and may still prefer web search over the AI SDK Assistant tools.
    </Note>
  </Tab>
</Tabs>

## When to Use the System Prompt

<Check>
  **Highly Recommended For:**

  - Complex multi-step SDK implementations
  - C# development (ensures C# code examples are retrieved)
  - Production code generation with proper error handling
  - Long development sessions requiring context consistency
</Check>

<Info>
  **Optional For:**

  - Quick one-off API lookups
  - Simple compatibility checks
  - Initial testing of AI SDK Assistant
</Info>

For more details on system prompt benefits, see [Tips: Using the System Prompt](/MCP-Server/tips#using-the-system-prompt-advanced).
