## CameraManager (Server)

Starts the native binary, polls for cameras, auto-connects, listens for SSE events, and reconnects on failure. One class replaces `ServerManager` + `CameraClient` + `EventStream`.

Import from `camera-remote-web-api/server` (Node.js only).

### Quick Start

```typescript
import { CameraManager } from 'camera-remote-web-api/server';

const manager = new CameraManager({ port: 8080 });

manager.on('camera-ready', async ({ cameraId, camera }) => {
  console.log(`${camera.model} connected`);

  // Set priority to PC Remote (required for remote control)
  await manager.client.setPriorityKey(cameraId, { setting: 'pc-remote' });

  // Take a photo
  await manager.client.triggerShutter(cameraId);
});

manager.on('camera-disconnected', ({ cameraId, error }) => {
  console.log(`Camera ${cameraId} disconnected`, error ?? '');
});

await manager.start();
// Binary boots → cameras discovered → auto-connected → 'camera-ready' fires
```

<Info>
`manager.client` is a full `CameraClient` instance — use it for all REST calls (properties, actions, live view, SD card).
</Info>

### Options

```typescript
const manager = new CameraManager({
  // Server options
  port: 8080,              // Default: 8080
  autoPort: true,          // Find next available port if busy
  binaryPath: '/path/to', // Explicit path to CameraWebApp binary

  // Connection options
  autoConnect: true,       // Auto-connect discovered cameras (default: true)
  connectionMode: 'remote', // Default mode: 'remote' | 'remote-transfer' | 'contents'
  autoReconnect: true,     // Reconnect on unexpected disconnect (default: true)
  maxReconnectAttempts: 5, // Give up after N attempts (default: 5)
  pollInterval: 5000,      // Camera discovery interval in ms (default: 5000)
});
```

### Lifecycle

```typescript
// Start — boots binary, begins polling, subscribes to SSE
await manager.start();

// Access the underlying client for REST calls
manager.client.setProperty(cameraId, 'iso', { value: '400' });

// Check binary status
await manager.isRunning();  // true
manager.getPort();          // 8080

// Stop — closes SSE, stops polling, kills binary
await manager.close();
```

### Additional Server Methods

```typescript
manager.getPort();           // Current port (may differ from requested if autoPort)
await manager.isRunning();   // Binary health check
manager.getStdout();         // Binary stdout lines
manager.getStderr();         // Binary stderr lines
manager.getPid();            // Binary process ID
```

## CameraManager (Browser)

Same API as the server variant, but connects to an **already-running** server instead of spawning a binary. Use this in React, Next.js client components, Electron renderers, or any browser context.

Import from `camera-remote-web-api` (the default export).

```typescript
import { CameraManager } from 'camera-remote-web-api';

const manager = new CameraManager({
  baseUrl: 'http://localhost:8080',
  autoConnect: true,
  connectionMode: 'remote',
});

manager.on('camera-ready', ({ cameraId, camera }) => {
  console.log(`${camera.model} ready`);
});

manager.start();  // Synchronous — just starts polling + SSE
```

<Warning>
The browser variant does **not** have `getPort()`, `isRunning()`, `getStdout()`, `getStderr()`, or `getPid()` — those are server-only methods that manage the native binary.
</Warning>

## Events

All CameraManager variants emit the same events.

| Event | Payload | When |
|-------|---------|------|
| `camera-found` | `{ camera }` | New camera detected by polling |
| `camera-lost` | `{ camera }` | Camera no longer detected (unplugged/offline) |
| `camera-connecting` | `{ cameraId }` | Connection attempt started |
| `camera-ready` | `{ cameraId, camera }` | Connected and ready for commands |
| `camera-disconnected` | `{ cameraId, error? }` | Disconnected (explicit or unexpected) |
| `connection-failed` | `{ cameraId, error, attempt }` | Connection attempt failed |
| `error` | `{ message }` | Manager-level error (poll failure, SSE error) |

```typescript
manager.on('camera-found', ({ camera }) => {
  console.log(`Discovered ${camera.model} (${camera.id})`);
});

manager.on('connection-failed', ({ cameraId, error, attempt }) => {
  console.log(`Attempt ${attempt} failed: ${error}`);
});

// Remove a specific listener
const handler = ({ cameraId }) => console.log('ready', cameraId);
manager.on('camera-ready', handler);
manager.off('camera-ready', handler);

// Remove all listeners for an event
manager.off('camera-ready');
```

## State Machine

Each camera tracked by CameraManager has a state:

```
detected ──→ connecting ──→ connected
   ↑              │              │
   │              ↓              ↓ (unexpected disconnect)
   └── detected ← connection-failed    disconnected
                                          │
                                          ↓ (autoReconnect)
                                     scheduleReconnect
                                          │
                                          ↓ (backoff: 2s, 4s, 6s... max 30s)
                                       detected
```

```typescript
// Inspect camera state at any time
const cameras = manager.getCameras();
// → [{ info: { id, model, connectionType, connected }, state: 'connected', reconnectAttempts: 0 }]

const cam = manager.getCamera('D10F60149B0C');
// → { info: {...}, state: 'connected', reconnectAttempts: 0 }
```

## Manual Connection Control

Disable auto-connect to manage connections yourself:

```typescript
const manager = new CameraManager({
  autoConnect: false,  // Don't connect on discovery
  autoReconnect: false,
});

manager.on('camera-found', async ({ camera }) => {
  // Connect only specific models
  if (camera.model === 'ILCE-9M3') {
    await manager.connect(camera.id, { mode: 'remote' });
  }
});

// Explicit disconnect
await manager.disconnect('D10F60149B0C');
```

## Multi-Camera

CameraManager handles multiple cameras automatically. Each camera gets independent state tracking and reconnection.

```typescript
const manager = new CameraManager({ connectionMode: 'remote' });

manager.on('camera-ready', async ({ cameraId, camera }) => {
  console.log(`${camera.model} ready`);
  await manager.client.setPriorityKey(cameraId, { setting: 'pc-remote' });
});

await manager.start();

// Later — trigger all connected cameras
const cameras = manager.getCameras().filter(c => c.state === 'connected');
await Promise.all(
  cameras.map(c => manager.client.triggerShutter(c.info.id))
);
```

## Next.js Pattern

Server boots the binary in an API route; browser uses CameraManager to connect.

<CodeGroup>
```typescript app/api/server/route.ts (Server)
import { CameraManager } from 'camera-remote-web-api/server';

let manager: CameraManager | null = null;

export async function POST() {
  if (!manager) {
    manager = new CameraManager({
      port: 8080,
      autoPort: true,
      binaryPath: '/path/to/CameraWebApp', // explicit path for Next.js bundler
    });
    await manager.start();
  }
  return Response.json({ port: manager.getPort() });
}
```

```typescript app/page.tsx (Browser)
'use client';
import { CameraManager } from 'camera-remote-web-api';
import { useEffect, useState } from 'react';

export default function CameraPage() {
  const [cameras, setCameras] = useState([]);

  useEffect(() => {
    // Boot the server
    fetch('/api/server', { method: 'POST' })
      .then(r => r.json())
      .then(({ port }) => {
        const manager = new CameraManager({
          baseUrl: `http://localhost:${port}`,
        });

        manager.on('camera-ready', () => {
          setCameras(manager.getCameras());
        });

        manager.start();
        return () => manager.close();
      });
  }, []);

  return <pre>{JSON.stringify(cameras, null, 2)}</pre>;
}
```
</CodeGroup>

<Warning>
Next.js bundler strips native `createRequire` — always pass an explicit `binaryPath` in API routes.
</Warning>

## Direct API Access

For full control without managed state, use the lower-level classes directly.

### CameraClient

Typed HTTP client for all REST endpoints.

```typescript
import { CameraClient } from 'camera-remote-web-api';

const client = new CameraClient('http://localhost:8080');
```

**Connection**
```typescript
const { cameras } = await client.listCameras();
await client.connectCamera(id, { mode: 'remote' });
await client.disconnectCamera(id);
await client.getConnectionStatus(id);
```

**Properties**
```typescript
await client.getProperty(id, 'iso');
await client.setProperty(id, 'iso', { value: '0x00000190' });  // ISO 400
await client.getAllProperties(id);
await client.setPriorityKey(id, { setting: 'pc-remote' });
```

**Actions**
```typescript
await client.triggerShutter(id);
await client.halfPress(id);
await client.afShutter(id);
await client.controlZoom(id, { direction: 'in', speed: 'normal' });
await client.stopZoom(id);
await client.focusNearFar(id, { step: 3 });
```

**Live View**
```typescript
await client.enableLiveView(id);
await client.startLiveViewStream(id);
const frame = await client.getLiveViewFrame(id);  // ArrayBuffer (JPEG)
await client.stopLiveViewStream(id);
```

**SD Card**
```typescript
const files = await client.listSDCardFiles(id, 1);  // slot 1
await client.downloadSDCardFile(id, 1, contentId, fileId);
```

**Settings**
```typescript
await client.getSaveInfo(id);
await client.setSaveInfo(id, { path: '/tmp/photos', prefix: 'IMG_', startNo: 1 });
```

### EventStream

Server-Sent Events client for real-time camera notifications.

```typescript
import { EventStream } from 'camera-remote-web-api';

// All cameras
const events = new EventStream('http://localhost:8080');

events.on('propertyChanged', (data) => console.log(data.codes));
events.on('downloadComplete', (data) => console.log(data.filename));
events.on('afStatus', (data) => console.log(data.state));
events.on('warning', (data) => console.log(data.code, data.message));
events.on('error', (data) => console.log(data.code));

// Specific camera
const camEvents = new EventStream('http://localhost:8080', cameraId);

// Clean up
events.close();
```

### LiveViewStream

WebSocket client for binary JPEG live view frames. Import from `camera-remote-web-api/server` (Node.js only).

```typescript
import { LiveViewStream } from 'camera-remote-web-api/server';

const liveView = new LiveViewStream('ws://localhost:8080', cameraId);
liveView.onFrame((jpeg: ArrayBuffer) => {
  // Render JPEG frame (~15fps)
});
liveView.close();
```

### ServerManager

Low-level binary process manager. Use CameraManager instead unless you need manual control over the binary lifecycle.

```typescript
import { ServerManager } from 'camera-remote-web-api/server';

const server = new ServerManager({
  port: 8080,
  autoPort: true,
});

await server.start();
await server.isRunning();  // true
server.getPort();          // 8080
await server.stop();
```

## Connection Modes

| Mode | Use Case |
|------|----------|
| `remote` | Full camera control + auto-transfer photos to host |
| `remote-transfer` | Full camera control + explicit SD card file pull |
| `contents` | SD card browsing and file download only |
