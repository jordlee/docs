## Server Status

`GET /api/server/status` — health check endpoint. Returns server version, uptime, platform, and camera counts. Used by `ServerManager` and `CameraManager` to confirm the server is ready.

<CodeGroup>
```bash curl
curl http://localhost:8080/api/server/status
```

```typescript TypeScript
await manager.isRunning(); // true — uses this endpoint internally
```

```json Response
{
  "success": true,
  "server": {
    "version": "3.0.0",
    "sdkVersion": "V2.01.00",
    "uptime": 42,
    "platform": "darwin-arm64"
  },
  "cameras": {
    "connected": 1,
    "discovered": 2
  }
}
```
</CodeGroup>

## Server Logs

`GET /api/server/logs` — returns buffered server log output. Useful for debugging without terminal access. Query params: `lines` (default 100), `level` (default `info`).

<CodeGroup>
```bash curl
curl http://localhost:8080/api/server/logs
```

```json Response
{
  "success": true,
  "logs": [
    {
      "timestamp": "2026-03-02T12:18:45",
      "level": "info",
      "message": "GET /api/server/status"
    }
  ],
  "total": 47
}
```
</CodeGroup>

## Shutdown

`POST /api/server/shutdown` — initiates graceful shutdown: disconnects all cameras, closes open SSE streams, then exits. `ServerManager.stop()` (in [`@alpha-sdk/api`](/web-api/server-package)) calls this before falling back to SIGTERM.

```bash
curl -X POST http://localhost:8080/api/server/shutdown
```
