This walks through the TypeScript / Node.js path. Python and Swift quickstarts live on their respective [SDK reference pages](/sdk/overview).

## 1. Initialize your project

If you don't already have a `package.json`:

```bash
npm init -y
npm pkg set type=module
```

<Info>
`@alpha-sdk/client` and `@alpha-sdk/api` ship dual ESM + CJS builds, but ESM is the recommended path on Node 18+. Setting `"type": "module"` enables `import`/`export` syntax.
</Info>

## 2. Install the SDK + server

Two packages: the [REST client](/sdk/typescript) and the [server distribution](/web-api/server-package), which ships the native server binary plus a Node.js `ServerManager` class for embedding the lifecycle.

```bash
npm install @alpha-sdk/client @alpha-sdk/api
```

The correct native binary for your current OS is pulled in automatically as an `optionalDependency`. On first run you'll be asked to accept the [Sony Camera Remote SDK License Agreement](https://support.d-imaging.sony.co.jp/app/sdk/licenseagreement_d/en.html).

## 3. Capture a photo

```ts
import { ServerManager } from "@alpha-sdk/api";
import { AlphaSDKClient } from "@alpha-sdk/client";

const server = new ServerManager();
await server.start();

const client = new AlphaSDKClient({
  environment: `http://localhost:${server.getPort()}`,
});

const { cameras } = await client.cameras.list();
const cameraId = cameras[0].id;

await client.cameras.connect({
  cameraId,
  mode: "remote-transfer",
  reconnecting: "on",
});

// Required for the host PC to drive most settings:
await client.properties.setPriorityKey({ cameraId, setting: "pc-remote" });

await client.actions.shutter({ cameraId });
console.log(`Photo captured on ${cameras[0].model}`);

await client.cameras.disconnect({ cameraId });
await server.stop();
```

## 4. Run it

Connect your Sony camera via USB, then:

```bash
npx tsx your-script.ts
```

You should see `Photo captured on ILCE-XXXX` and a new file on the camera's SD card.
