## 1. Install and start the server

Install [`@alpha-sdk/api`](/web-api/server) — it ships the native server binary, the `camera-server` CLI, and a programmatic `ServerManager` class:

```bash
npm install -g @alpha-sdk/api
```

The matching platform binary (`@alpha-sdk/darwin-arm64`, `@alpha-sdk/linux-x64`, `@alpha-sdk/linux-arm64`, `@alpha-sdk/win32-x64`) is pulled in automatically — no separate download. On first run you'll be prompted to accept the [Sony Camera Remote SDK License Agreement](https://support.d-imaging.sony.co.jp/app/sdk/licenseagreement_d/en.html).

Start the camera server:

```bash
camera-server start
```

The server runs on port 8080 and discovers cameras connected via USB or network automatically.

```bash
camera-server status            # check what's running
camera-server stop              # graceful shutdown
camera-server                   # interactive TUI (no args)
```

<Note>
  Prefer a different language? Install the matching client too — `camera-server install-sdk ts` (TypeScript), `camera-server install-sdk python` (Python), or `camera-server install-sdk swift` (SwiftPM snippet). See the [SDK reference](/sdk/overview).
</Note>

## 2. Discover Cameras

```bash
curl http://localhost:8080/api/cameras
```

Note the camera `id` from the response — you'll use it in all subsequent calls.

## 3. Connect

```bash
curl -X POST http://localhost:8080/api/cameras/{id}/connection \
  -H "Content-Type: application/json" \
  -d '{"mode": "remote"}'
```

## 4. Take Control

```bash
curl -X PUT http://localhost:8080/api/cameras/{id}/priority-key \
  -H "Content-Type: application/json" \
  -d '{"setting": "pc-remote"}'
```

<Warning>Priority key **must** be set to `pc-remote` before the camera accepts remote commands.</Warning>

## 5. Shoot

```bash
curl -X POST http://localhost:8080/api/cameras/{id}/actions/af-shutter
```

## 6. Disconnect

```bash
curl -X DELETE http://localhost:8080/api/cameras/{id}/connection
```
