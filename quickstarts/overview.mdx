## Available platform entry points

Most developers should start with the Alpha Camera REST platform and choose the client or tool that fits their stack.

| Product | Language | What it does |
|---------|----------|--------------|
| [TypeScript Client SDK](/sdk/typescript) | TypeScript / JS | Official REST client. Pair with the [server distribution](/web-api/server-package) for an embedded server lifecycle. |
| [Python Client SDK](/sdk/python) | Python | Sync and async REST client with Pydantic models. |
| [Swift Client SDK](/sdk/swift) | Swift | Native `async/await` client for iOS, macOS, tvOS, and watchOS. |
| [Alpha Camera REST API](/web-api/server-package) | Any | The HTTP and SSE server itself. Install via npm; ships with the `camera-server` CLI and a Node.js `ServerManager` class. |
| [MCP Servers](/mcp-server/overview) | — | Connect Claude, Cursor, and other assistants to the SDK docs, Help Guides, and Web API docs. |

## How they relate

```
Your application
    │
    ├─ @alpha-sdk/client  (TypeScript)  ┐
    ├─ alpha-sdk-client    (Python)      ├─► Alpha Camera REST API (HTTP + SSE on :8080)
    ├─ AlphaSDK            (Swift)       ┘             │
    └─ raw HTTP client     (any lang)  ─►              │
                                                        │
                                            Native server binary
                                            (shipped via @alpha-sdk/api,
                                             built on Sony's Camera Remote SDK)
                                                        │
                                                Sony camera (USB / network)
```

All three managed clients call the same REST and SSE server. The server is distributed on npm as `@alpha-sdk/api` — run it standalone via `camera-server start`, or, from a Node application, embed its lifecycle via the `ServerManager` class.
